# Общая функциональность системы
## Регистрация

Пользователь вводит поля:
- `login`
- `name`
- `surname`
- `last_name`
- `password`

Ему пристраивается дата регистрации registered_at
И также он может выбрать картинку, уйдет запрос к `images` и `images->id —- users->photo_id`

## Авторизация
Пользователь вводит поля:
- `login`
- `password`

На бекенде происходит проверка:
- Такой пользователь существует
- Хеш полученного пароля совпадает с хешем в базе

Возвращается токен авторизации с которым пользователь ходит дальше

## Создание группы
Пользователь вводит поля:
- `name`
- `description`
- `is_open`
- `max_participant_count`

Получает далее id группы
Теперь связь `groups->creator_id —- users->id`

## Вступление в группу
- Если группа открыта:
    - Пользователь отправляет groups->id группы в которую хочет вступить
    - Далее он попадает в участники группы в таблицу `group_participants`
    - Теперь `id` этого пользователя в `group_participants->user_id`
    - Теперь `id` этой группы в таблице `group_participants -> group_id`

- Если группа закрыта:
    - Пользователь отправляет запрос на создание запроса на вступление в группу,
    - В запросе указывает `id` группы
    - Далее ему возвращается объект из таблицы `group_enter_request`

Создатель группы получает список объектов из таблицы `group_enter_request` в которых `group_id` это та группа у которой поле `creator_id` - это `id` пользователя, те смотрит запросы в свои группы
Пользователь который смотрит свои запросы в другие группы, идет просто фильтр в `group_enter_request` по `user_id`

Как только админ группы посылает запрос на approved, id пользователя который отправил запрос, те из `group_enter_request -> user_id`  заносится в `group_participant -> user_id`, и также `id` группы

## События

События это некоторые занятия которые создаются с некоторой периодичностью. В них указывается название, описание и максимальное число участников.

При создании события, сначала, создается объект периодичности (таблица `rapids`) в котором будет указано как часто это событие будет повторяться. 

Далее, у пользователя в календаре рисуются все события с учетом периодичности.
При нажатии на «будущее» занятие, или текущее, его необходимо «начать», чтобы информация о нем перенеслась в таблицу `event_participants`.

В эту таблицу плодятся записи по всем участникам группы (таблицы `group_participants`) по связи `event_participants->participant_id —- group_participant->id`
А также для конкретного события `event_participants->event_id —- events->id`

Фильтруя записи по `participant_id` можно получить статистику присутствия или оплаченных занятий внутри одной группы
Фильтруя записи по `event_id` можно получить всех участников одного конкретного занятия и заносить по каждому из них необходимую информацию(присутствие, оплата и прочее)

Далее пользователь может посмотреть в календаре свое расписание по занятием внутри группы
Или статистику по всем своим уже прошедшим занятиям.

## Слоты

Функционал схож с группами и событиями, только тут нет как таковых групп

Создатель слота может создать событие (слот) в котором также может указать периодичность повторение слота (если это необходимо, может и разово), описание, название. (Таблица `slots`)

Далее пользователи могут вступать в эти слоты сами и смотреть свою статистику по всем слотам в которым они принимали участие (`slot_participants`)

Если эту таблицу `slot_participants` фильтровать по `slot_id` то можно посмотреть кто заявлен на это событие, сделать заметки о посещении и оплате
Если фильтровать по `user_id` то можно посмотреть в каких слотах этот пользователь принимал участие и его статистику.
